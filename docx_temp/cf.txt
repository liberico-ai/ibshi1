1.2. Sáu điểm đau (pain points) từ cách làm hiện tại
PAIN 1 — Phương án tài chính là Excel 97 sheet, 32 cột. Khi PM cập nhật dự toán, Treasury phải copy-paste sang Dòng tiền trừ lùi bằng tay → thường sai lệch 2–5% giữa 2 sheet.
PAIN 2 — Mỗi drawdown (Vay) / liquidation (Giải tỏa) phải gõ thủ công vào 1 file Excel riêng, upload cho VPBank. Năm 2025 riêng dự án 095 có ≥25 file vay + 10 file giải tỏa. Không có audit trail ai tạo, ai duyệt.
PAIN 3 — &quot;Chi tiết thanh toán các đợt&quot; (sheet trong Phương án) đang là 258 dòng giao dịch gõ tay, phân cột sang 11 cost bucket (Thép / Vật tư phụ / Vật liệu hàn / Sơn / …). Không reconcile tự động với Misa hay IBS One&apos;s PurchaseOrder/Invoice.
PAIN 4 — Dòng tiền trừ lùi (còn phải chi theo tháng từ cut-off tới cuối dự án) đang làm Excel thủ công hàng tháng. Không ai biết khi nào hết tiền mua vật tư trước khi Treasury chạy số.
PAIN 5 — Bảng tổng hợp nợ vay cấp công ty 2026 phải gom từ 7 file bank rời. Số &quot;Còn phải thu của KH&quot; và &quot;Dư nợ NH&quot; không link tự động với doanh thu / invoice → rủi ro double-count hoặc miss khoản.
PAIN 6 — Không có realtime view: &quot;Nếu VOGT 094 delay 2 tuần thì VOGT 095 có đủ tiền vật tư U2 không?&quot; — hiện không trả lời được trong ngày.
1.3. Cơ hội tự động hoá
Folder 095 là dự án xuất khẩu trung bình (169,8 tỷ). IBS HI hiện có 28 dự án active (theo IBS One). Nếu pattern 095 này được automate và áp dụng cho 28 dự án, Treasury team sẽ tiết kiệm ≈4–6 giờ/dự án/tháng × 28 = 112–168 giờ/tháng (≈0,7–1,0 FTE). Ngoài ra, 3 bàn thanh toán nhanh hơn (approval drawdown trong giờ thay vì ngày) sẽ cải thiện DSO từ 173 → mục tiêu ≤90 ngày.
2. Feature Overview
2.1. Vị trí trong IBS One Platform
Project Cashflow Feature nằm trong bounded context &quot;Treasury &amp; Cashflow&quot; (đã định nghĩa trong FA Platform Design PHẦN D.1). Feature extends 2 model có sẵn trong ibshi1 (CashflowEntry, Budget) và bổ sung 10 model mới chuyên trách cho nghiệp vụ dòng tiền dự án + quản lý khoản vay ngân hàng.

Model A.1 — ProjectFinancePlan
model ProjectFinancePlan {
  id                String    @id @default(cuid())
  projectId         String
  version           Int       @default(1)
  cutoffDate        DateTime  @db.Timestamptz  // &quot;-ĐC - 31 03 2026&quot;
  status            PlanStatus @default(DRAFT)  // DRAFT / APPROVED / SUPERSEDED

  // Contract summary (from sheet &quot;Dòng tiền và dự toán gốc&quot;)
  customerId        String?
  scopeDescription  String?   @db.Text
  contractValueVnd  Decimal   @db.Decimal(18, 2)
  contractValueCcy  String?   // USD / EUR
  contractValueFx   Decimal?  @db.Decimal(15, 2)
  paymentTermDays   Int       // NET 45
  grossMarginPct    Decimal   @db.Decimal(5, 4)

  // Audit
  createdById       String
  approvedById      String?
  approvedAt        DateTime?
  createdAt         DateTime  @default(now())  @db.Timestamptz

  project           Project   @relation(fields: [projectId], references: [id])
  budgetLines       ProjectBudgetLine[]
  monthlyCashflow   ProjectCashflowMonthly[]
  milestones        ContractPaymentMilestone[]

  @@unique([projectId, version])
  @@index([status, cutoffDate])
}
Model A.2 — ProjectBudgetLine (hierarchical; map from 3 sheets DT03/DT05/DT6)
model ProjectBudgetLine {
  id              String   @id @default(cuid())
  planId          String
  parentId        String?  // hierarchical: A → 1 → 1.1
  code            String   // &quot;VTC01&quot;, &quot;PC&quot;, &quot;DVT&quot; — mã dự toán từ Excel
  sectionType     BudgetSection  // MATERIAL / LABOUR / SUBCONTRACT / OVERHEAD / GUARANTEE / VAT / CONTINGENCY
  itemName        String
  unit            String?  // kg / m2 / chuyến
  quantity        Decimal? @db.Decimal(18, 4)
  unitPrice       Decimal? @db.Decimal(18, 4)
  amountVnd       Decimal  @db.Decimal(18, 2)
  percentOfTotal  Decimal? @db.Decimal(5, 4)

  // Optional dimensional links (allow fact-based allocation)
  materialId      String?  // from ibshi1 Material master
  activityCode    String?  // from BOM (Tekla operation)
  costCenterId    String?

  plan            ProjectFinancePlan @relation(fields: [planId], references: [id])
  parent          ProjectBudgetLine? @relation(&quot;Hierarchy&quot;, fields: [parentId], references: [id])
  children        ProjectBudgetLine[] @relation(&quot;Hierarchy&quot;)

  @@index([planId, sectionType])
  @@index([planId, code])
}

enum BudgetSection {
  MATERIAL
  LABOUR
  SUBCONTRACT
  OVERHEAD
  GUARANTEE
  VAT
  CONTINGENCY
}

Model B.1 — CreditFacility (group-level, 7 rows for IBS HI)
model CreditFacility {
  id              String   @id @default(cuid())
  bankId          String
  facilityNo      String   // &quot;Số:BCLC-4314-01&quot;
  facilityType    FacilityType   // SHORT_TERM_LOAN / GUARANTEE / LONG_TERM_LOAN / LEASING
  signedDate      DateTime @db.Timestamptz
  expiryDate      DateTime @db.Timestamptz
  tenorMonths     Int      // 12, 36, 240

  loanLimit       Decimal  @db.Decimal(18, 2)  // Hạn mức cho vay
  guaranteeLimit  Decimal  @db.Decimal(18, 2)  // Hạn mức BL
  totalLimit      Decimal  @db.Decimal(18, 2)  // Tổng

  interestRateVnd Decimal? @db.Decimal(5, 4)   // %/năm VND
  interestRateUsd Decimal? @db.Decimal(5, 4)   // %/năm USD

  collateralNotes String?  @db.Text  // &quot;Tín chấp 70% quyền thu&quot;, &quot;TSĐB bằng 50%&quot;
  isActive        Boolean  @default(true)

  bank            Bank     @relation(fields: [bankId], references: [id])
  loanContracts   LoanContract[]

  @@index([bankId, facilityType, isActive])
}

7.3. Edge Cases &amp; Failure Modes
E1 — Vendor không có bank account trong master: UI buộc Treasury nhập + save vào Vendor master trước khi tạo Drawdown line (không cho &quot;one-off&quot; BA).
E2 — Invoice amount khác với số tiền cho vay: allow, nhưng tổng amountFundedVnd của tất cả drawdowns không được vượt invoiceAmount. System hard-check.
E3 — Drawdown bị bank reject: status REJECTED → trigger alert cho Treasury + CFO; auto-create follow-up task
E4 — Partial disburse (bank chỉ giải 80%): system ghi amountFunded theo thực tế; flag variance &gt; 5%
E5 — LoanContract expire nhưng vẫn có outstanding: alert Treasury 30 ngày trước, 15 ngày, 7 ngày, 1 ngày
E6 — Credit facility utilization &gt; 90%: alert CFO + block new drawdowns pending CFO over-ride
E7 — Excel format VPBank thay đổi: config template trong code (không hard-code) để swap template mới nhanh
E8 — Multi-currency: drawdown USD/EUR convert sang VND bằng FxRate (pull VCB Rate API) tại thời điểm execution
7.4. Compliance touchpoints
TT200 — hạch toán vay: Dr 112 / Cr 341 (ngắn hạn) hoặc 341-LT (dài hạn). Lãi vay: Dr 635 / Cr 335
VAS 16 — Chi phí đi vay: capitalize vào 154 (WIP) nếu là vay cho sản xuất dự án specific — cần config per loan
VAS 10 — Ảnh hưởng của tỷ giá hối đoái: FX reval monthly cho LoanContract USD/EUR outstanding
ISA 315 — SoD matrix 7.2 phải được enforce bằng RBAC ở schema level (không chỉ UI)
